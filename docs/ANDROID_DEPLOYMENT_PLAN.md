# ANDROID DEPLOYMENT & PACKAGING PLAN
## WIBAWA NUSANTARA — PWA to Native Android App (TWA/Capacitor)

> [!IMPORTANT]
> **CSS & Responsive Assurance Policy:**
> Never compile or build an Android APK/AAB until the web app's mobile layout is 100% responsive, overflow-free, and notch-safe. A TWA or Capacitor shell renders the exact CSS/HTML output of the PWA. If there is a horizontal overflow or styling bug on the website, it will be visible in the compiled app.

---

## 1. Chosen Technology Stack: Trusted Web Activities (TWA)

We utilize **Trusted Web Activities (TWA)** via the Google-developed **Bubblewrap CLI** as our primary packaging mechanism.

### Why TWA?
* **Full Native Look**: Hides browser navigation bars, rendering in full screen standalone mode.
* **Lightweight Footprint**: The APK is typically less than 3MB because it leverages the system Chrome engine.
* **Always Up-to-Date**: Changes to the Next.js frontend are automatically reflected in the app without republishing to the Play Store.
* **Chrome Feature Support**: Inherits top-tier V8 Javascript engine speed, autofill capabilities, and credential management.

### Alternative Stack: Capacitor
For future releases needing hardware integrations (e.g. low-level camera APIs or background sync), we can transition to **Capacitor** by running:
```bash
npm install @capacitor/core @capacitor/cli
npx cap init "Wibawa Nusantara" "com.wibawanusantara.app" --web-dir=out
npx cap add android
```

## 2. Technical Setup using Bubblewrap CLI

Follow these steps to generate a release-ready Android App Bundle (`.aab`) and debugging APK (`.apk`):

### Step A: Prerequisites & Tooling
Ensure your development computer has the following tools installed:
1. **Node.js** (v18+)
2. **Java Development Kit (JDK 17)**
3. **Android SDK Command-line Tools** (including `apksigner` and `zipalign`)

### Step B: Install Bubblewrap
Install the Google-maintained Bubblewrap CLI globally:
```bash
npm install -g @bubblewrap/cli
```

### Step C: Initialize the Android Project
Run the initialization wizard against our live production domain:
```bash
bubblewrap init --manifest=https://wibawa-nusantara.com/manifest.webmanifest
```
*Bubblewrap will read our PWA manifest (`manifest.webmanifest`), configure package name (`com.wibawanusantara.app`), define start URL (`/`), and import high-res app icons.*

### Step D: Build the Packages
To compile the package, run:
```bash
bubblewrap build
```
During the build, Bubblewrap will prompt you to:
1. Create a new secure signing **Keystore**.
2. Specify passwords for the Keystore and key alias.
3. Output the following files inside your build directory:
   - `app-release-signed.aab` (Upload this file to the **Google Play Console**).
   - `app-debug.apk` (Use this for local testing/sideloading on physical Android devices).

## 3. Digital Asset Links — Domain Verification

To hide the Chrome address bar and confirm ownership of the domain:

1. Retrieve the **SHA-256 fingerprint** of the signing key generated by Bubblewrap:
   ```bash
   keytool -list -v -keystore android.keystore -alias android
   ```
2. Create a domain verification file named `assetlinks.json`:
   ```json
   [
     {
       "relation": ["delegate_permission/common.handle_all_urls"],
       "target": {
         "namespace": "android_app",
         "package_name": "com.wibawanusantara.app",
         "sha256_cert_fingerprints": [
           "XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX"
         ]
       }
     }
   ]
   ```
3. Upload this file to your Next.js project public folder so it resolves exactly at:
   `https://wibawa-nusantara.com/.well-known/assetlinks.json`

## 4. Local Testing & Verification Checklist

Before publishing to the Google Play Store, perform the following validation tests on actual Android hardware:

| Test Scenario | Validation Steps | Expected Behavior |
| :--- | :--- | :--- |
| **Horizontal Stability** | Sideload the `app-debug.apk` and scroll through pages (Feed, Checkout, Marketplace). | Zero horizontal layout shifts. Viewport width remains locked to exactly `100vw`. |
| **Safe Area Compatibility** | Open the app on a device with a screen notch/island. | Top header fits below the system status icons. Bottom navigation buttons remain clear of the navigation pill. |
| **Authentication Flow** | Log in, log out, and trigger email verification. | Cookies and session tokens persist inside the Android TWA shell correctly. |
| **Realtime Operations** | Create a listing, update an order status, or initiate a chat. | Live Supabase subscriptions receive data instantly, and WebPush notifications trigger device-level alerts. |

## 5. Security & Keystore Policy

> [!CAUTION]
> **Signing Key Security Warning:**
> Never commit the keystore file (`*.keystore` or `*.jks`) or its passwords to GitHub or public version control. If you lose this key, you will be unable to update the application on the Google Play Store, requiring a brand-new package release. Backup the key securely in a password manager.
